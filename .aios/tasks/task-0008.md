# Provide an operator dashboard view of the loop

## Objective

Give the human operator one screen that answers "what are the agents doing
and what needs me": a generated, self-contained HTML dashboard of every
Task, its lifecycle position, and its evidence.

## Acceptance Criteria

- `aios dashboard [--root <path>] [--out <file>]` generates a single HTML
  file (default `dashboard.html` at the repository root) and prints the
  written path; the existing `run` subcommand's behavior, output, and exit
  codes are unchanged, and `--help` plus the README document the new
  subcommand.
- The dashboard shows an overview header (project, Task counts per state,
  generation time) and one card or row per Task with: id, title, a
  state badge visually distinct per state, `retry.count`/`retry.limit`,
  the `approval` field, `last_review` id and verdict, and the number of
  recorded Attempts.
- A Task in `state: approval` whose decision file
  `.aios/approvals/<task-id>` does not exist is prominently flagged as
  awaiting the human decision, including the exact file path to write.
- The latest Review's findings for each reviewed Task are readable on the
  dashboard (a collapsed/expandable section is acceptable).
- Tasks and Reviews are loaded with the same strict validation the engine
  uses; a document that fails to load renders as a visible error entry
  naming the document and reason instead of aborting the whole dashboard.
- The HTML is fully self-contained and offline: inline CSS only, no
  external requests, no JavaScript required for the core view.
- Generation is a one-shot read-only pass: no server, no watcher, no
  daemon; `.aios/` documents are never modified, and the generated file is
  added to `.gitignore`.
- The data-collection and rendering logic is exported and covered by
  `node --test` cases under `test/` (no browser needed); `npm test` passes
  for the whole repository.

## Constraints

- No new runtime dependencies.
- Do not change the Loop Engine's transition or validation behavior and do
  not modify the v1 document contracts; adding read-only listing helpers
  is acceptable.
- First version only: no live refresh, no git history mining, no charts,
  no theming options.
- Repository documents and dashboard text are in English, matching the
  repository convention.

## Context

Operators currently reconstruct loop state from raw `.aios/` files and git
history. The repository already contains everything the screen needs:
Tasks under `.aios/tasks/` (front matter is machine-owned truth), Reviews
under `.aios/reviews/`, human decisions under `.aios/approvals/`, and the
store/validation API in `src/documents.js` (`TaskStore`, used by
`src/cli.js`). `.aios/tasks/README.md` defines states and invariants;
task-0007 is intentionally `blocked` and task-0006 `done` with
`approval: approved`, which the dashboard should render faithfully.
