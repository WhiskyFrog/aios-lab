# Provide a Human approver Worker

## Objective

Provide a command Worker that resolves the `approver` Role to a human: it
turns an operator-written decision file into an `aios.result/v1` approval
Result, and halts the run with clear instructions while the human has not
decided yet.

## Acceptance Criteria

- `workers/human-approver.mjs` implements the command Worker contract for
  the `approver` Role: Task document on stdin, `AIOS_TASK_ID` and
  `AIOS_ROLE` in the environment, exactly one `aios.result/v1` JSON object
  on stdout.
- The worker reads the human decision from the file
  `.aios/approvals/<task-id>` relative to its working directory. A file
  containing exactly `approved` or `rejected` (surrounding whitespace
  ignored) yields a `status: success` Result with that decision.
- When the decision file does not exist, the worker returns a
  `status: failure` Result whose reason names the exact file path to create
  and the two accepted contents, so the engine halts without a Task
  transition and the operator can resume after deciding.
- Any other file content yields a `status: failure` Result that quotes the
  invalid content.
- An `AIOS_ROLE` other than `approver`, or a missing `AIOS_TASK_ID`, is
  unusable input: the worker exits nonzero with a stderr diagnostic.
- The worker never writes to the repository; it only reads.
- The decision-reading logic is exported and covered by `node --test` cases
  under `test/` that use a temporary directory and run via `npm test`
  without any engine run or Claude session.
- README documents the Human approver: the decision file protocol and the
  halt-then-resume flow for `approval: required` Tasks.
- `npm test` passes for the whole repository.

## Constraints

- Do not modify anything under `src/`, the v1 document contracts, or
  `workers/claude-worker.mjs`.
- Do not modify `.aios/` documents; the decision file is written by the
  human operator, never by this worker.
- No new runtime dependencies; plain Node 22, single self-contained script.

## Context

The Sprint 0 workflow reserves an optional approval gate between Reviewer
and Done: a Task with `approval: required` enters `state: approval` after a
passing review, and its Assignment decides who approves. This Task makes
that worker a human bridge. This Task itself carries `approval: required`
and the Assignment already maps `approver` to the worker being built here,
so its own final gate will exercise the new worker: the run should halt
awaiting a human decision until the operator writes
`.aios/approvals/task-0006`. The command Worker contract and envelope
shapes are specified in `.aios/results/README.md`; `workers/claude-worker.mjs`
shows an existing adapter and `test/claude-worker.test.js` the test style.
