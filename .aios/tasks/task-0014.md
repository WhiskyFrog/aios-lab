# Plan the AIOS product dashboard

## Objective

Turn the product goal “build a web dashboard that introduces AIOS and shows
current and upcoming work” into a reviewed, profile-aware plan of focused
`aios.task/v1` proposals under `plans/aios-dashboard-app/`.

## Acceptance Criteria

- `PLAN.md` selects the narrowest registered Planner profile, records why it
  fits, and explains how the profile shaped the decomposition.
- The plan preserves the existing self-contained `aios dashboard` foundation
  unless it gives a concrete reason that the product goal requires a server or
  separate application architecture.
- The proposed product covers an AIOS introduction and workflow explanation,
  current Task lifecycle data, adopted upcoming Tasks, not-yet-adopted plan
  proposals, clear next actions, and responsive/accessibility quality.
- Data-source and UI work are separated where that makes each proposal
  independently implementable and verifiable in one Worker session; shared
  foundations precede their consumers in `PLAN.md` execution order.
- Every proposal is a valid placeholder `aios.task/v1` document with observable
  acceptance criteria, default retry state, no Attempts, and no references to
  other proposal placeholders in its body.
- `node src/cli.js adopt plans/aios-dashboard-app --check` passes without
  modifying `.aios/tasks/`, and the Implementer reports that command in
  Verification.
- The Reviewer evaluates both mechanical validity and whether the proposed
  Tasks collectively deliver the stated dashboard rather than merely restating
  its pages.

## Constraints

- This is a planning Task. Write only under `plans/aios-dashboard-app/`; do not
  implement the dashboard and do not modify `.aios/`, source, tests, or git.
- Do not add orchestration, live polling, a daemon, or a backend unless the
  plan explicitly demonstrates that the goal cannot be met by extending the
  current one-shot self-contained dashboard.
- Keep Task ordering in `PLAN.md`; do not add dependency fields to proposals.

## Context

The repository already has `src/dashboard.js`, `test/dashboard.test.js`, and a
README Dashboard section for a one-shot offline HTML view of Tasks, Reviews,
approvals, and Worker sessions. The newly approved Planner convention and
profile catalog are documented in the README Planner section. The missing
product surface is a polished AIOS introduction plus visibility into both
adopted future Tasks and proposals still waiting under `plans/`.
